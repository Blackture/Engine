Mathematics

In mathematics, there are many variants to calculate a result for the same goal. This engine implements many approaches to achieve the same result. Why different approaches may be needed is described per approach.

Rotation Approaches

There are many rotation approaches to apply a specific rotation. Commonly, one calculates an angle and applies the angle, so does this engine too. There are three types of rotations implemented: angle rotation, rotation matrices, and quaternions. 

  • Rotation Matrix
  • Quaternion Rotation
  • Angle Rotation

Rotation Matrix

Poimora has two different types of rotation matrices. Those types are 2x2 and 3x3 rotation matrices. It has a pre-built 45° 2x2 rotation matrix.

The problem of using rotation matrices is the gimbal lock. In case, switch to quaternion rotation.

API Reference (2x2)API Reference (3x3)

Quaternion Rotation

Poimora does not support quaternion rotation yet. (7/1/2023 - 1:47 PM (PDT))

The problem of using quaternions is that they aren't implemented yet.

API Reference

Angle Rotation

Poimora supports to get the angle between many objects (like Vectors, Planes, …). This means, one can get an angle and apply it onto another object in many scenarios.

The problem of using angles is that they need a lot of computation which can lead to worse performance, lags, or crashes.

Projection Approaches

There are many rotation approaches to apply a specific rotation. Commonly, one calculates an angle and applies the angle, so does this engine too. There are three types of rotations implemented: angles, rotation matrices, and quaternions. 

  • Orthographic Projection
  • Perspective Projection
  • Projection for Glasses
  • Oblique Projection
