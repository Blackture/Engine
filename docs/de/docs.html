Mathematik

In der Mathematik gibt es viele varianten, wie man eine Lösung für ein bestimmtes Ziel berechnen kann. Diese Engine implementiert viele Lösungswege, um dasselbe Ziel zu erreichen. Warum verschiedene Lösungsansätze möglicherweise benötigt werden, ist per Ansatz beschrieben.

Rotationskonzepte

Es gibt viele Möglichkeiten, um eine bestimmte Rotation durchzuführen. Üblicherweise berechnet man einen Winkel und verwendet diesen dann, sowie auch diese Engine. Es sind 3 Arten von Rotationen implementiert: Winkel-Rotationen, Rotationsmatrizen und Quaternionen.

  • Rotationsmatrix
  • Quaternionen-Rotation
  • Winkel-Rotation

Rotation Matrix

Poimora hat 2 Arten von Rotationmatrizen. Diese Typen sind 2x2 und 3x3 Rotationmatrizen. Es hat eine vorgefertigte 45° bzw. π/4 2x2 Rotationsmatrix.

Das Problem des Nutzens von Rotationsmatrizen ist der Gimbal Lock. Wechsle in diesem Fall zur Quaternionen-Rotation.

Quaternion Rotation

Poimora verfügt bisher noch nicht über Quaternionen-Rotationen. (7/1/2023 - 1:47 PM (PDT))

Das Problem des Nutzens von Quaternionen ist, dass sie noch nicht implementiert sind.

Winkel-Rotation

Poimora ermöglicht die Ermittlung des Winkels zwischen vielen Objekten (wie Vektoren, Ebenen, …). Das heißt, man kann einen Winkel ermitteln und ihn in vielen Szenarien auf ein anderes Objekt applizieren.

Das Problem des Nutzens von Winkeln ist, dass sie eine Menge Berechnungen benötigen, was zu schlechterer Leistung, Lags (Verzögerungen) oder Abstürzen führen kann.

Projektionskonzepte

There are many rotation approaches to apply a specific rotation. Commonly, one calculates an angle and applies the angle, so does this engine too. There are three types of rotations implemented: angles, rotation matrices, and quaternions.
